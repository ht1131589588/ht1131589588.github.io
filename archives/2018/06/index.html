项目结构

  • build （打包后资源目录）
  • public （公用模板）
  • src （主要内容目录）
    • api （api借口）
    • assets （资源文件目录，js，css，图片）
    • components （公用组件）
    • config （配置文件）
    • pages （页面文件）
    • router （路由配置）
  • .gitignore
  • package.json

语法规范

  1. 文件夹目录小写
  2. 驼峰命名-首字母小写
  3. 命名用英文全屏，如果太长可以适当缩写一部分
